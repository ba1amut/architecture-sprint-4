
# Проблемы
## Производительность MES
Долгая загрузка дашборда с заказами.
Фильтрация и пагинация не решают проблему.
Операторы не могут оперативно видеть новые заказы, что влияет на их вознаграждение.
## Обработка заказов
Задержки в выполнении заказов.
Клиенты жалуются на просроченные заказы.
Увеличение нагрузки после открытия API.
## API для внешних продавцов
Внешние продавцы жалуются на отсутствие заказов.
Потеря крупных контрактов из-за проблем с обработкой заказов.
## Процессы разработки и поддержки продукта
Задержки релизов из-за ручного тестирования.
Отсутствие масштабируемости.
Недостаток специалистов для решения текущих задач

# Решения

## Процессы разработки и поддержки продукта
Автоматизация тестирования внедрение CI/CD
Мониторинг
Масштабирование
Балансировка нагрузки
Найм дополнительных специалистов(DevOps  - для настройки текущей инфраструктуры, QA - для настройки автоматического тестирования, Backend-разработчик для оптимизации производительности системы.)
## Производительность MES
Оптимизация базы данных;
Кэширование;
Асинхронная загрузка;
Масштабирование базы данных.

## Обработка заказов
Приоритезация заказов;
Оптимизация RabbitMQ;
Масштабирование системы;
Мониторинг и аналитика.

## API для внешних продавцов
Подтверждение заказов;
Ограничение нагрузки;
Уведомления для внешних продавцов о статусе их заказов.

# Целевая архитектура
### 1. Онлайн-магазин (B2C)

- **Планируемые работы**:
    - Оптимизация интерфейса для ускорения загрузки страниц.
    - Внедрение ленивой загрузки для улучшения производительности.
    - Добавление уведомлений для клиентов о статусе заказов.
- **Планируемое состояние**:
    - Высокая производительность даже при увеличении нагрузки.
    - Улучшенный пользовательский опыт.
### 2. CRM

- **Планируемые работы**:
    - Оптимизация запросов к базе данных.
    - Внедрение кэширования данных (Redis).
    - Добавление системы приоритетов для заказов.
- **Планируемое состояние**:
    - Быстрая обработка заказов даже при высокой нагрузке.
    - Улучшенная интеграция с MES.

### 3. MES (Manufacturing Execution System)

- **Планируемые работы**:
    - Оптимизация базы данных (индексы, шардирование).
    - Внедрение асинхронной обработки заказов.
    - Улучшение интерфейса для операторов.
- **Планируемое состояние**:
    - Высокая производительность даже при сложных 3D-моделях.
    - Удобный интерфейс для операторов.
### 4. MES API

- **Планируемые работы**:
    - Внедрение защиты от перегрузки.
    - Добавление подтверждения получения заказов.
    - Улучшение документации API.
- **Планируемое состояние**:
    - Надёжное взаимодействие с внешними продавцами.
    - Защита от перегрузки системы.
### 5. RabbitMQ
- **Планируемые работы**:
    - Увеличение количества воркеров для обработки сообщений.
    - Настройка логики для обработки ошибок.
- **Планируемое состояние**:
    - Высокая надёжность и производительность.
### 6. База данных
- **Планируемые работы**:
    - Шардирование базы данных:
	- Использование реплик для чтения. 
    - Оптимизация запросов (индексы, кэширование).
- **Планируемое состояние**:
    - Высокая производительность даже при увеличении нагрузки.
    - Масштабируемость.
### 7. 3D Files Storage
- **Планируемые работы**:
    - Оптимизация хранения (например, использование облачных решений).
    - Внедрение сжатия файлов для экономии места и трафика.
- **Планируемое состояние**:
    - Быстрый доступ к файлам.
    - Экономия ресурсов.
### 8. Облачная инфраструктура
- **Планируемые работы**:
    - Внедрение балансировки нагрузки.
- **Планируемое состояние**:
    - Высокая отказоустойчивость и масштабируемость.
### 9. Мониторинг и аналитика
- **Планируемые работы**:
    - Внедрение Prometheus и Grafana для сбора и визуализации метрик.
    - Настройка алертинга для оперативного реагирования на проблемы.
- **Планируемое состояние**:
    - Прозрачность работы системы.
    - Быстрое выявление и устранение проблем.
### 10. CI/CD конвейер
- **Планируемые работы**:
    - Автоматизация тестирования (unit, integration, E2E).
    - Внедрение автоматического деплоя в release и prod после успешного прохождения тестов.
    - Использование инструментов для автоматизации (например, Jenkins, GitLab CI/CD).
- **Планируемое состояние**:
    - Быстрый и надёжный процесс выпуска новых версий.
    - Минимизация ручного вмешательства.

# Три первых пункта по решению проблемы

Для успешного улучшения архитектуры и решения текущих проблем компании рекомендуется начать с выполнения следующих трёх ключевых пунктов:
 1. Оптимизация базы данных и внедрение шардирования решит ключевую проблему с производительностью MES, что критично для операторов и клиентов.
 2. Внедрение мониторинга и аналитики позволит выявить другие узкие места в системе и оперативно реагировать на проблемы.
 3. Автоматизация CI/CD конвейера ускорит процесс разработки и улучшит качество выпускаемых версий, что важно для дальнейшего масштабирования.

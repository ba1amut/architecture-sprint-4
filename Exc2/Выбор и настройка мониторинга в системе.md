# Мотивация
Внедрение мониторинга и аналитики позволит выявить другие узкие места в системе и оперативно реагировать на проблемы.

# Выбор подхода к мониторингу
### Предложения по мониторингу для каждого компонента

#### 1. Интернет-магазин (Internet Shop)
- **Подход**: RED.
- **Метрики**:
    - **Rate**: Количество запросов от клиентов (например, загрузка 3D-моделей, создание заказов).
    - **Errors**: Количество ошибок при загрузке страниц или отправке заказов.
    - **Duration**: Время загрузки страниц и выполнения операций.
- **Инструменты**: Prometheus, Grafana.
#### 2. Shop API
- **Подход**: RED.
- **Метрики**:
    - **Rate**: Количество запросов к API (например, получение списка товаров, сохранение заказов)
    - **Errors**: Количество ошибок при обработке запросов.
    - **Duration**: Время выполнения запросов.
- **Инструменты**: Prometheus, Grafana.
#### 3. CRM и CRM API
- **Подход**: RED.
- **Метрики**:
    - **Rate**: Количество запросов на управление заказами и клиентами.
    - **Errors**: Количество ошибок при обработке запросов.
    - **Duration**: Время выполнения операций (например, создание заказа, обновление статуса).
- **Инструменты**: Prometheus, Grafana.
#### 4. MES и MES API
- **Подход**: «Четыре золотых сигнала».
- **Метрики**:
    - **Latency**: Время расчёта стоимости изделий и обработки заказов.
    - **Traffic**: Количество заказов, поступающих в MES.
    - **Erros**: Количество ошибок при расчёте стоимости или обработке заказов.
    - **Saturation**: Загрузка системы (например, CPU, память).
- **Инструменты**: Prometheus, Grafana.
#### 5. Базы данных (Shop DB, MES DB)
- **Подход**: USE.
- **Метрики**:
    - **Utilization**: Использование CPU, памяти, дисков.
    - **Saturation**: Количество активных соединений, длина очереди запросов.
    - **Errors**: Количество ошибок при выполнении запросов.
- **Инструменты**: Prometheus, Grafana, специализированные инструменты для мониторинга PostgreSQL.
#### 6. Очередь сообщений (RabbitMQ)
- **Подход**: USE.
- **Метрики**:
    - **Utilization**: Использование ресурсов (CPU, память).
    - **Saturation**: Длина очереди сообщений, время обработки сообщений.
    - **Errors**: Количество ошибок при обработке сообщений.
- **Инструменты**: Prometheus, Grafana, RabbitMQ Exporter.
#### 7. Хранилище 3D-файлов (3D Files Storage)

- **Подход**: USE.
- **Метрики**:
    - **Utilization**: Использование дискового пространства.
    - **Saturation**: Количество одновременных операций чтения/записи.
    - **Errors**: Количество ошибок при загрузке или извлечении файлов.
- **Инструменты**: Prometheus, Grafana, S3-совместимые инструменты мониторинга.
### Общий подход к мониторингу

1. **Централизованный сбор метрик**:
    - Использовать **Prometheus** для сбора метрик со всех компонентов системы.
    - Настроить экспортеры для каждого компонента (например, RabbitMQ Exporter, PostgreSQL Exporter).
2. **Визуализация и алертинг**:
    - Использовать **Grafana** для визуализации метрик.
    - Настроить алертинг для уведомления о критических проблемах (например, высокий уровень ошибок, перегрузка ресурсов).
3. **Логирование**:
    - Внедрить централизованное логирование (например, ELK Stack или Loki) для анализа ошибок и проблем.
4. **Трассировка**:
    - Внедрить распределённую трассировку (например, Jaeger) для отслеживания выполнения запросов между компонентами.
